# 検査シフト自動作成アプリ (SIFT Scheduler)

心電図検査 (ECG) とエコー検査のシフトを **OR-Tools CP-SAT ソルバー** で自動作成する Streamlit アプリです。患者枠は最大 30 人まで可変で扱えます。

---

## クイックスタート

### 必要なもの

- Python 3.9 以上（推奨 3.11）
- pip

### セットアップ

```bash
# 仮想環境を作成して有効化（Windows）
python -m venv .venv
.venv\Scripts\activate

# macOS / Linux
python3 -m venv .venv
source .venv/bin/activate

# 依存パッケージをインストール
pip install -r requirements.txt

# 起動
streamlit run app.py
```

ブラウザで `http://localhost:8501` が開きます。

---

## 使い方

- **初回利用**: まず `スタッフ設定` を確認し、そのまま `シフト作成` へ進みます
- **前回の続き / 当日キャンセル再最適化**: `シフト作成` タブ先頭の **アプリデータの復元** からバックアップ JSON を読み込みます

### 1. 入力タブ — 当日の情報を入力する

| 入力項目 | 説明 |
|---------|------|
| **休みスタッフ** | 当日休みの人を選択 |
| **女性患者枠** | 女性患者が入る枠番号を指定 |
| **キャンセル枠** | キャンセルになった枠番号を指定 |
| **当番** | 生体①・生体②・立ち上げ・バックアップ・転送・早朝エコーを割り当て |
| **シフト時間変更** | 午前休・午後休・時短など、スタッフごとに勤務開始/終了を変更。昼当番候補に含めるかも選択可能（デフォルト: 含めない） |
| **心臓研修枠** | 心臓研修の対象枠を指定 |
| **固定担当** | 特定の患者枠だけ担当者を事前固定 |
| **患者枠メモ** | 女性対応希望・難しめなどの申し送りを入力 |
| **朝フォロー / 夕方フォロー** | フォロー業務の拘束時間を通常検査に反映 |
| **スタッフごとの当日補正** | 今日だけ軽め / 多めにしたいスタッフへ補正を入れる |
| **エコー開始遅延時の負荷軽減** | 遅い時間からしかエコーに入れない人の最大負荷を自動で少し下げる |
| **昼当番** | シフト作成後に候補から選択（事前に固定しない方式）。判定窓や候補条件は制約設定で変更可能 |

### 2. 最適化の実行

「**シフト作成**」ボタンを押すと 3 段階のソルバーが順番に実行されます。

1. **STEP 1 — 条件どおり**: すべてのハード制約を適用し、ECG 連続系も厳密に確認
2. **STEP 2 — 休憩引き直し**: 当番条件は維持したまま、休憩候補を組み直して再探索
3. **STEP 3 — 最終条件**: さらに当番のシフト時間制約を外し、負荷条件を中心に最終探索

いずれかのステップで解が見つかれば結果を表示し、採用されたステージは結果画面に表示されます。

### 3. 結果の確認

- **シフト表**: 枠ごとの心電図・エコー担当を表示
- **Excel 貼付用テーブル**: コピー＆ペーストで Excel に転記できる表
- **スタッフ別サマリー**: 担当領域数・2 人担当件数・休憩時間帯
- **制約違反レポート**: 違反があれば内容と原因を表示
- **担当者ガント**: 入れ替え・編集・当日キャンセル再最適化を実施

### 4. 保存と復元

- **下書き保存**: 入力内容を一時保存
- **テンプレート保存**: 繰り返し使うパターンを保存
- **保存履歴**: 過去の最適化結果を閲覧・復元
- **結果を保存**（各タブ共通）: 保存履歴に追加し、同時に **バックアップ JSON** のダウンロードボタンが表示されます。このファイルにはスタッフ設定・下書き・テンプレート・保存履歴・スケジュール結果がすべて含まれます
- **アプリデータの復元**（シフト作成タブ先頭）: バックアップ JSON を読み込んで、上記すべてをまとめて復元します

> **⚠️ 重要:** CSV・Excel・HTML のダウンロードは**印刷・閲覧専用**です。
> 当日キャンセル再最適化など、保存したスケジュールを後から編集する機能を使うには、
> 必ず **「結果を保存」でバックアップ JSON をダウンロード** してください。

### 5. 当日キャンセル再最適化

前日までに作成・保存したスケジュールに対して、当日キャンセルが発生した際にスケジュールを組み直す機能です。

#### 前日までの準備

1. シフトを作成し結果を確認する
2. いずれかのタブで「**結果を保存**」を押す（保存履歴への追加＋バックアップ JSON ダウンロードが同時に行われます）
3. 表示される「**バックアップ JSON をダウンロード**」ボタンを押してファイルを保管する

#### 当日の操作

1. **スケジュールを復元する**
   - シフト作成タブ先頭の「**アプリデータの復元**」にバックアップ JSON を読み込む
   - または保存履歴タブ → 該当日の履歴を選んで「**この結果を復元**」
2. 担当者ガントタブの「**当日キャンセル再最適化**」を展開
3. **再最適化範囲**を設定する
   - **開始枠**: まだ実施していない最初の枠（これより前の枠は実施済みとして固定）
   - **終了枠**: 再最適化の最後の枠（デフォルトは末枠）
4. **キャンセル枠**をタップ/クリックして選択する
   - 再最適化範囲の外（実施済み範囲）のキャンセルも選択可能（実施されなかったものとして扱われます）
5. 「**再最適化を実行**」→ プレビューで負荷差分・公平性スコアを確認
6. 「**再最適化結果を反映**」で確定、または「**破棄**」でやり直し

> **💡 ヒント:** CSV や Excel、HTML ダウンロードからはスケジュールを読み込めません。
> 印刷用ファイルとは別に、必ず「結果を保存」→「バックアップ JSON をダウンロード」で保管してください。

## スタッフ設定

`スタッフ設定` タブで各スタッフの属性を管理します。

| 設定項目 | 説明 |
|---------|------|
| **勤務時間** (`shift_start` / `shift_end`) | その人の勤務時間帯 |
| **最大負荷** (`max_load`) | 1 日に割り当てる最大領域数 |
| **理想負荷** (`ideal_load`) | 目標とする領域数 |
| **休憩時間** (`break_minutes`) | 連続休憩の必要分数（デフォルト 65 分） |
| **休憩分割** (`allow_split_break`) | 休憩を 2 分割してよいか（デフォルト: 許可） |
| **エコー可能領域** | 腹部・心臓・頸動脈・体表・下肢動脈・下肢静脈など |
| **見学領域** | 研修中で見学が必要な領域 |

設定は `staff_config.json` に保存されます。
